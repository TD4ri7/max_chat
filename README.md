# MAX Chat

MAX Chat — это веб-приложение для общения, которое позволяет пользователям взаимодействовать друг с другом в режиме реального времени[cite: 3]. Приложение поддерживает глобальный чат, приватные диалоги и групповые чаты[cite: 3]. Проект разработан с учетом принципов распределенных приложений[cite: 3].

**Составитель:** Димитрий Марковский[cite: 3]  
**Учебная группа:** JPTV23[cite: 3]

## Функции
- **Управление пользователями:** Регистрация, безопасный вход (на основе сессий) и выход из системы[cite: 3].
- **Управление профилем:** Изменение личных данных и загрузка аватара[cite: 3].
- **Чаты:** Общая комната (глобальный чат), приватные беседы (1-на-1) и групповые чаты[cite: 3].
- **Сообщения:** Отправка текстовых сообщений и изображений в реальном времени[cite: 3].
- **Модерация:** Пользователи с правами администратора могут удалять неприемлемые сообщения[cite: 3].

## Технологии
- **Сервер (Backend):** Node.js, Express.js[cite: 3]
- **База данных:** MongoDB (Mongoose)[cite: 3]
- **Клиент (Frontend):** JavaScript (ES6+), HTML5, CSS3, Fetch API, FormData[cite: 3]
- **Аутентификация:** На основе сессий (Session-based Authentication)[cite: 3]

## Установка и запуск

### Предварительные требования
Убедитесь, что на вашем компьютере установлены:
- [Node.js](https://nodejs.org/) (версии 18.x или новее)[cite: 3]
- [MongoDB](https://www.mongodb.com/) (локальная служба или кластер MongoDB Atlas)[cite: 3]

### Пошаговое руководство

1. **Клонируйте репозиторий:**
```bash
   git clone <url-вашего-репозитория>
   cd max-chat
   ```[cite: 3]

2. **Установите необходимые зависимости:**
```bash
   npm install
   ```[cite: 3]

3. **Настройте переменные окружения:**
   Создайте в корневой папке проекта файл с именем `.env` и добавьте в него следующие конфигурации:
```env
   PORT=3000
   MONGODB_URI=mongodb://localhost:27017/maxchat
   SESSION_SECRET=minusalajanevõti123
   ```[cite: 3]

4. **Запустите приложение:**
   В режиме разработки (development):
```bash
   npm run dev
npm start
   ```[cite: 3]

5. **Откройте приложение в браузере:**
   Перейдите по адресу `http://localhost:3000`[cite: 3]

## Конечные точки REST API

| Метод | URL | Описание и данные |
| :--- | :--- | :--- |
| **POST** | `/api/auth/register` | Регистрация нового пользователя (username, email, password)[cite: 3] |
| **POST** | `/api/auth/login` | Вход пользователя (создание сессии)[cite: 3] |
| **POST** | `/api/auth/logout` | Выход из системы (завершение сессии)[cite: 3] |
| **GET** | `/api/auth/me` | Запрос данных активного пользователя на основе сессии[cite: 3] |
| **POST** | `/api/profile/avatar` | Загрузка аватара профиля (в виде объекта FormData)[cite: 3] |
| **GET** | `/api/chats/dialogs` | Загрузка списка активных чатов пользователя[cite: 3] |
| **POST** | `/api/messages` | Отправка сообщения или изображения в конкретный чат[cite: 3] |
| **GET** | `/api/messages/:id` | Загрузка хронологической истории сообщений чата[cite: 3] |
| **DELETE** | `/api/messages/:id` | Удаление сообщения (требуются права Admin или статус автора)[cite: 3] |

## Модели данных

### User (Пользователь)
- `_id`: ObjectId (Уникальный идентификатор)[cite: 3]
- `username`: String (Уникальное имя пользователя)[cite: 3]
- `email`: String (Уникальный email)[cite: 3]
- `password`: String (Хешированный пароль)[cite: 3]
- `about`: String (Краткое описание "о себе")[cite: 3]
- `avatar`: String (Путь к файлу изображения)[cite: 3]
- `role`: String (Роль в системе: `"user"` или `"admin"`)[cite: 3]

### Chat (Чат)
- `_id`: ObjectId[cite: 3]
- `type`: String (Тип чата: `"global"`, `"private"` или `"group"`)[cite: 3]
- `participants`: Array (Ссылки на модель User — участники)[cite: 3]
- `title`: String (Название группового чата)[cite: 3]
- `createdAt`: Date (Время создания)[cite: 3]

### Message (Сообщение)
- `_id`: ObjectId[cite: 3]
- `sender`: ObjectId (Ссылка на модель User — отправитель)[cite: 3]
- `chatId`: ObjectId (Ссылка на модель Chat)[cite: 3]
- `text`: String (Текст сообщения)[cite: 3]
- `imageUrl`: String (Путь к изображению, если оно прикреплено)[cite: 3]
- `createdAt`: Date (Временная метка отправки)[cite: 3]

## Безопасность
- **Управление сессиями:** Используется аутентификация на основе сессий для безопасного сохранения состояния[cite: 3].
- **Шифрование данных:** Пароли пользователей хешируются перед сохранением в базу данных[cite: 3].
- **Защита от XSS:** Валидация входящих данных на стороне клиента и очистка спецсимволов HTML[cite: 3].
- **Контроль файлов:** Строгая проверка типа и размера загружаемых изображений[cite: 3].
